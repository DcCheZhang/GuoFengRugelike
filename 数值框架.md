## 一、属性枚举（程序接口定义）

### 1.1 单位属性接口（TypeScript）

```typescript
// 单位基础属性接口（我方单位和敌方单位共用）
interface UnitAttributes {
  // === 核心属性 ===
  hp: number;             // 生命值（当前）
  maxHp: number;         // 生命值（最大）
  attack: number;        // 攻击力
  defense: number;       // 防御力
  
  // === 战斗属性 ===
  attackSpeed: number;   // 攻速（秒/次，值越小越快）
  moveSpeed: number;     // 移速（像素/秒）
  range: number;        // 射程（像素，近战=0）
  
  // === 特殊属性 ===
  critRate: number;     // 暴击率（0-1，如0.1=10%）
  critDamage: number;    // 暴击伤害倍数（如2.0=200%）
  dodgeRate: number;    // 闪避率（0-1）
  toughness: number;    // 韧性（减少被暴击率）
  
  // === 状态属性 ===
  isAlive: boolean;     // 是否存活
  reviveCountdown: number; // 复活倒计时（回合数，-1表示不倒计时）
}

// 单位等阶枚举
enum UnitTier {
  NORMAL = "normal",     // 普通（白）
  SCARCE = "scarce",    // 稀少（绿）
  LEGENDARY = "legendary", // 传奇（紫）
  MYTHICAL = "mythical"  // 神话（橙）
}

// 单位类型枚举
enum UnitType {
  WARRIOR = "warrior",     // 战士
  MAGE = "mage",           // 法师
  TANK = "tank",           // 坦克
  ASSASSIN = "assassin",   // 刺客
  SUMMONER = "summoner",   // 召唤
  SUPPORT = "support",      // 辅助
  CONTROL = "control"       // 控制
}

// 我方单位接口（扩展）
interface PlayerUnit extends UnitAttributes {

  tier: UnitTier;
  type: UnitType;
  level: number;          // 等阶等级（1-5）
  skills: Skill[];        // 技能列表
  artifacts: Artifact[];  // 装备的法宝列表

// 敌方单位接口（扩展）

  id: string;
  name: string;
  enemyType: EnemyType;   // 敌人类型（普通/精英/Boss）

  aiType: AIType;         // AI类型
}
```

### 1.2 战斗计算接口

```typescript
// 伤害计算结果接口
interface DamageResult {
  damage: number;        // 实际伤害值
  isCrit: boolean;       // 是否暴击
  isDodged: boolean;     // 是否被闪避
  isKilled: boolean;      // 是否击杀
}

// 战斗上下文接口（用于计算）
interface BattleContext {
  attacker: UnitAttributes;
  defender: UnitAttributes;
  skillCoefficient: number; // 技能系数（如1.0=100%攻击力）
  additionalEffects: Effect[]; // 附加效果（如灼烧、中毒）
}

// 效果接口（如灼烧、中毒、眩晕）
interface Effect {
  type: EffectType;
  value: number;         // 效果值
  duration: number;      // 持续时间（秒，-1表示永久）
  tickRate: number;      // 触发频率（秒/次）
}
```

## 二、战斗公式

### 2.1 伤害计算公式

```
伤害 = Max(1, (攻击方.攻击 - 防御方.防御 × 防御系数) × 技能系数 × 随机波动 × 暴击倍数)
```

**详细说明**：
- `防御系数`：默认0.8（防御的80%有效）
- `随机波动`：0.9 - 1.1 之间的随机数
- `暴击倍数`：如果暴击，则为 `暴击伤害倍数`（默认2.0）；否则为1.0
- `Max(1, ...)`：保证至少造成1点伤害

**TypeScript实现**：

```typescript
function calculateDamage(context: BattleContext): DamageResult {
  const { attacker, defender, skillCoefficient } = context;
  
  // 1. 检查闪避
  if (Math.random() < defender.dodgeRate - attacker.toughness * 0.01) {
    return { damage: 0, isCrit: false, isDodged: true, isKilled: false };
  }
  
  // 2. 计算基础伤害
  const defenseFactor = 0.8;
  let damage = Math.max(1, 
    (attacker.attack - defender.defense * defenseFactor) 
    * skillCoefficient 
    * (0.9 + Math.random() * 0.2) // 随机波动0.9-1.1
  );
  
  // 3. 检查暴击
  const isCrit = Math.random() < attacker.critRate;
  if (isCrit) {
    damage *= attacker.critDamage;
  }
  
  // 4. 应用伤害
  const actualDamage = Math.min(damage, defender.hp);
  defender.hp -= actualDamage;
  
  return {
    damage: actualDamage,
    isCrit,
    isDodged: false,
    isKilled: defender.hp <= 0
  };
}
```

### 2.2 暴击逻辑

**暴击率计算**：
```
最终暴击率 = 攻击方.暴击率 - 防御方.韧性 × 0.01
```

**暴击伤害**：
```
暴击伤害 = 基础伤害 × 攻击方.暴击伤害倍数
```

**示例**：
- 攻击方暴击率：15%
- 防御方韧性：50
- 最终暴击率：15% - 50×0.01 = 10%
- 暴击伤害倍数：2.0（200%）

### 2.3 闪避逻辑

**闪避率计算**：
```
最终闪避率 = 防御方.闪避率 - 攻击方.命中率（暂定）
```

**说明**：
- 暂时不实现"命中率"属性，默认命中率=1.0（100%命中）
- 所以 `最终闪避率 = 防御方.闪避率`

**闪避结果**：
- 如果闪避成功，伤害=0，显示"MISS"

## 三、属性成长公式

### 3.1 合成后属性成长

**公式**：
```
新属性 = 原属性 × (1 + 0.5 × 新阶数)
```

**示例**：
- 原单位：1阶普通，攻击=15
- 3个1阶普通合成 → 1个2阶普通
- 新阶数 = 2
- 新攻击 = 15 × (1 + 0.5 × 2) = 15 × 2 = 30

**各阶成长倍数**：
| 目标阶数 | 成长倍数 |
|-----------|-----------|
| 2阶 | × 2.0 |
| 3阶 | × 2.5 |
| 4阶 | × 3.0 |
| 5阶 | × 3.5 |

### 3.2 等阶属性差异

**公式**：
```
等阶属性倍数：
- 普通（白）：× 1.0
- 稀少（绿）：× 2.0
- 传奇（紫）：× 4.0
- 神话（橙）：× 8.0
```

**示例**：
- 基础单位（普通1阶）：攻击=15，防御=8，生命=120
- 稀少版（稀少1阶）：攻击=30，防御=16，生命=240
- 传奇版（传奇1阶）：攻击=60，防御=32，生命=480
- 神话版（神话1阶）：攻击=120，防御=64，生命=960

## 七、程序实现建议

### 7.1 属性计算优化

**建议1：使用缓存**
- 单位属性可能被频繁计算（如法宝加成、技能加成）
- 建议在属性变更时重新计算，而不是每帧计算

**建议2：使用事件系统**
- 属性变更时触发事件（如"OnAttackChanged"）
- 法宝/技能监听事件，应用加成

**示例（TypeScript + 事件系统）**：

```typescript
class Unit {
  private _attack: number;
  private baseAttack: number;
  private attackBuffs: Buff[] = [];
  
  get attack(): number {
    // 实时计算（考虑Buff）
    let finalAttack = this.baseAttack;
    for (const buff of this.attackBuffs) {
      finalAttack += buff.value;
    }
    return finalAttack;
  }
  
  addBuff(buff: Buff) {
    this.attackBuffs.push(buff);
    // 触发事件
    EventSystem.emit("OnAttackChanged", this);
  }
}
```

### 7.2 战斗计算优化

**建议1：使用对象池**
- 战斗中可能频繁创建/销毁DamageResult对象
- 建议使用对象池复用对象

**建议2：批量计算**
- 如果多个单位同时攻击同一目标，可以批量计算伤害
